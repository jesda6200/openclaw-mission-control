3. Ou installer depuis un dépôt Git (méthode manuelle)

     mkdir -p ~/.openclaw/skills
     git clone <url-du-repo> ~/.openclaw/skills/<nom-du-skill>
   - Vérifie qu’un SKILL.md est présent à la racine du répertoire cloné.

4. Redémarrer le service OpenClaw (pour recharger les skills)
   - openclaw gateway restart
   - Après redémarrage, OpenClaw détecte les nouveaux skills et les expose.

5. Vérifier / configurer
   - Lire SKILL.md du skill installé pour voir les étapes spécifiques, dépendances et variables d’environnement.
   - Certains skills demandent une configuration (API keys, secrets) ; range-les dans le fichier de config ou via les variables d’environnement sécurisées.
   - Si le skill fournit une commande d’activation dans openclaw config, suis ses instructions (consulte la doc du skill).

- Préfère l’installation via paquet officiel quand elle existe (npm) plutôt que copier manuel, pour les mises à jour automatiques.
