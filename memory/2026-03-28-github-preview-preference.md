# 2026-03-28 — Préférence prévisualisation GitHub

- Quand l’utilisateur demande une publication sur GitHub, ajouter par défaut une vraie prévisualisation de type “README vitrine”, comme sur `jesda6200/loyalty-mobile-app`.
- Le format attendu est prioritairement :
  - bannière / hero visuel en haut du README si pertinent
  - une ou plusieurs sections visuelles centrées (`assets/` ou dossier équivalent)
  - au minimum un showcase produit/app/interface réellement lisible sur la page GitHub
  - si utile, un schéma d’architecture ou un visuel complémentaire
- Ne plus se contenter d’une micro-image technique ou d’un placeholder discret quand le repo est publié sur le GitHub de l’utilisateur.
- Si le projet a une UI réelle existante, la preview doit venir du vrai projet.
- Les visuels synthétiques ne doivent servir que s’il n’existe pas encore d’interface montrable.
- Cette préférence s’applique par défaut aux futures publications GitHub, sauf indication contraire explicite.
